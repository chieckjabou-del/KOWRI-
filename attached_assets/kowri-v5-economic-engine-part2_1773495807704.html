Architecture Ledger du coffre

Le coffre utilise un compte Ledger de type CLIENT_WALLET avec un identifiant dédié (CLIENT_COFFRE_{user_id}_{goal_id}). Les fonds sont physiquement dans le Ledger. L'état "verrouillé" est logique : la State Machine refuse les débits tant que unlock_conditions n'est pas satisfait. Le déverrouillage = transaction Ledger DEBIT coffre → CREDIT wallet principal, via le flow Core Engine complet.

// F1 · FLUX ÉCONOMIQUES DÉTAILLÉS
Interactions Modules × Ledger — Écritures Réelles

Flux A — Payout Tontine Standard (M01 + Core Engine)

-- Payout 500 000 XOF · Tranche 6% · Abonnement GOLD (-1%) → taux net = 5%
GROSS_PAYOUT     = 500_000 XOF
RATE_BPS         = 600 - 100 = 500 bps  (6% - 1% discount abonnement)
COMMISSION       = 500_000 × 500 / 10_000 = 25_000 XOF
NET_PAYOUT       = 475_000 XOF

-- Écriture Ledger atomique (3 lignes, une seule transaction DB) :
DEBIT  CLIENT_TONTINE_POOL[tontine_id]    500_000 XOF
CREDIT CLIENT_WALLET[beneficiary_id]      475_000 XOF
CREDIT PLATFORM_FEES                       25_000 XOF
-- Vérification : 500_000 = 475_000 + 25_000 ✓ (invariant I2)

-- commission_calculations.status → COMMITTED · ledger_tx_id référencé
-- Notification : "Vous recevez 475 000 XOF (net après commission 5%)"

Flux B — Revente Position Marché Secondaire (M07 + M08)

-- Position vendue 140 000 XOF · Comm. KOWRI 3% · Comm. organisateur 0.5%
SALE_PRICE    = 140_000 XOF
KOWRI_COMM    = 140_000 × 300 / 10_000 = 4_200 XOF
ORG_COMM      = 140_000 × 50  / 10_000 =   700 XOF
NET_TO_SELLER = 135_100 XOF

-- Ledger (4 lignes, atomique) :
DEBIT  CLIENT_WALLET[buyer]        140_000 XOF
CREDIT CLIENT_WALLET[seller]       135_100 XOF
CREDIT PLATFORM_FEES                 4_200 XOF
CREDIT CLIENT_WALLET[organizer]        700 XOF
-- 140_000 = 135_100 + 4_200 + 700 ✓
-- position_transfers.status → COMPLETED · tontine_members mis à jour

Flux C — Micro-crédit avec remboursement (M17)

-- Crédit 50k XOF · 30j · 5% intérêt · 2% origination · Score 640
-- Décaissement :
DEBIT  CLIENT_CREDIT_ESCROW       51_000 XOF  (50k + 1k frais)
CREDIT CLIENT_WALLET[borrower]    50_000 XOF
CREDIT PLATFORM_FEES               1_000 XOF  ← 51_000 = 50_000 + 1_000 ✓

-- Remboursement J+30 :
DEBIT  CLIENT_WALLET[borrower]    52_500 XOF  (50k + 2.5k intérêts)
CREDIT CLIENT_CREDIT_ESCROW       50_000 XOF
CREDIT PLATFORM_FEES               2_500 XOF  ← 52_500 = 50_000 + 2_500 ✓
-- Score M16 recalculé : +15 pts (remboursement à temps)

Flux D — Coffre-fort avec déverrouillage anticipé (M20)

-- Dépôt coffre : 30 000 XOF · Déverrouillage anticipé : pénalité 2%
-- Dépôt (DEBIT wallet, CREDIT coffre) :
DEBIT  CLIENT_WALLET[user]        30_000 XOF
CREDIT CLIENT_COFFRE[user][goal]  30_000 XOF  ← 30_000 = 30_000 ✓

-- Déverrouillage anticipé (pénalité 2% = 600 XOF) :
PENALTY = 30_000 × 200 / 10_000 = 600 XOF
DEBIT  CLIENT_COFFRE[user][goal]  30_000 XOF
CREDIT CLIENT_WALLET[user]        29_400 XOF
CREDIT PLATFORM_FEES                 600 XOF  ← 30_000 = 29_400 + 600 ✓

Flux E — Crowdfunding All-or-Nothing (M30)

-- Campagne objectif 500k · Clôture : 520k collectés · Frais plateforme 4%
-- Chaque pledge : DEBIT wallet bailleur → CREDIT escrow campagne
DEBIT  CLIENT_WALLET[backer]      50_000 XOF
CREDIT CLIENT_CROWDFUND_ESCROW    50_000 XOF  ← × 10+ bailleurs jusqu'à 520k

-- Objectif atteint → Déblocage final (un seul appel Core Engine) :
PLATFORM_FEE = 520_000 × 400 / 10_000 = 20_800 XOF
DEBIT  CLIENT_CROWDFUND_ESCROW    520_000 XOF
CREDIT CLIENT_WALLET[creator]     499_200 XOF
CREDIT PLATFORM_FEES               20_800 XOF  ← 520_000 = 499_200 + 20_800 ✓

-- Si objectif NON atteint → remboursement individuel de chaque pledge
-- DEBIT escrow → CREDIT wallet bailleur (× N transactions, idempotent)

✓ Architecture KOWRI V5.0 — Complète

L'Economic Engine Expansion transforme KOWRI en infrastructure financière communautaire africaine complète. Le Core Engine V4.x reste intouchable. Les 30 modules s'étendent en 6 couches hiérarchiques connectées au Ledger central. La progression de tontine locale → super-app panafricaine est architecturalement cohérente, financièrement saine, et réglementairement tracée. Chaque franc CFA passe par le Ledger. Aucun état financier n'existe en dehors de lui.
